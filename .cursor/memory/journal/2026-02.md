## 2026-02-28

- [Quests][RTL][QuestMapFrame][Fix] Keep quest title icon/decorations stable across EN/AR toggle cycles
  - Why: On toggle paths, `QuestInfoTitleHeader` can briefly expose a plain EN title (without leading decoration). Re-extracting from that plain text cleared glyph/link state, so the Arabic title icon disappeared on the next toggle-on.
  - What changed:
    - Added per-quest `Quests.Details._TitleDecorCache` to persist decorated title metadata (`title`, `tags`, `glyph`, `linkRef`, `linkText`).
    - In `TranslateOn`, when live extraction returns no decoration, reuse cached decoration for the current quest instead of clearing state.
    - In `TranslateOff`, prefer cached decorated EN title when available so EN/AR toggles keep the same title decoration baseline.
    - Follow-up hardening:
      - Treat leading glyph decorations (`!`, `?`, non-ASCII glyphs) as decorated titles during EN title upgrades in `QuestPrepare` (previously only `|H`/`|T`/`|A` were considered).
      - Normalize title decor/font cache keys to `tostring(questID)` to avoid numeric-vs-string key misses.
      - Added verbose debug lines for title decoration resolution/source on `TranslateOn`/`TranslateOff`.
      - Prevent title icon font cache overwrite from Arabic `__post__` passes; only capture icon source font from non-Arabic header text.
      - In icon render path, reject cached `WOWTR_Font1` and fall back to `Original_Font1` to keep glyph decorations visible.
      - Reset icon position locks on non-`__post__` passes and on `TranslateOff` to avoid stale overlay coordinates after width/anchor restore.
      - Tightened RTL overlay offsets (`QuestInfoTitleHeader`: `+0`, `QuestProgressTitleText`: `+4`) to reduce out-of-frame clipping risk.
      - Added runtime RTL icon X clamping based on parent-right boundary and icon rendered width (`GetStringWidth`) to prevent any right-edge overflow.
      - Added explicit RTL title right padding (`8px`) and matched icon clamp right-edge padding (`8px`) so overlay remains comfortably inside the header frame.
  - Key files:
    - `common/Quests/Details.lua`
